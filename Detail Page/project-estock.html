Overview

Latar Belakang

Pengelolaan stok barang di gudang masih sering dilakukan secara manual menggunakan catatan fisik atau spreadsheet yang tidak real-time. Hal ini menyebabkan ketidakakuratan data stok, kesulitan melacak barang masuk dan keluar, serta proses audit yang memakan waktu lama.

E-Stock hadir sebagai solusi digital berbasis mobile yang memungkinkan pengelola gudang untuk memantau dan mengelola stok kapan saja dan di mana saja — langsung dari genggaman tangan.

Design Process

Proses Pengembangan

01
Analisis Kebutuhan & User Story
Mendefinisikan kebutuhan sistem berdasarkan skenario penggunaan nyata di lingkungan gudang. Membuat user story untuk setiap peran pengguna — admin gudang dan petugas lapangan — dan memetakan alur kerja mereka sehari-hari.
User StoryRequirements AnalysisUse Case
02
Perancangan Database & Arsitektur
Merancang struktur database untuk entitas utama: Produk, Kategori, Transaksi Masuk, Transaksi Keluar, dan Riwayat. Memastikan relasi antar tabel memungkinkan laporan stok yang akurat dan mudah diambil.
Database DesignEntity RelationshipSystem Architecture
03
UI Design di Figma
Merancang antarmuka yang bersih dan fungsional dengan fokus pada kemudahan scan visual — daftar stok harus langsung bisa dibaca dengan cepat, status stok rendah ditandai dengan warna menonjol, dan aksi utama mudah dijangkau dengan ibu jari.
FigmaMobile UIComponent DesignColor System
04
Implementasi React Native
Mengimplementasikan semua halaman dan fitur CRUD menggunakan React Native. Memastikan performa scrolling yang smooth saat daftar stok panjang, dan transisi antar halaman yang responsif.
React NativeState ManagementLocal StorageNavigation
05
Testing & Quality Assurance
Melakukan pengujian fungsional untuk setiap operasi CRUD, memastikan tidak ada data yang hilang atau korup. Pengujian dilakukan di beberapa perangkat Android dengan spesifikasi berbeda untuk memastikan kompatibilitas.
Functional TestingUATBug FixingCross-Device

Yang Saya Pelajari

Proyek E-Stock mengajarkan saya pentingnya kesederhanaan dalam UI untuk aplikasi operasional. Berbeda dengan aplikasi consumer yang mengejar estetika, aplikasi gudang harus mengutamakan kecepatan dan kejelasan — pengguna tidak punya waktu untuk mempelajari interface yang rumit ketika sedang sibuk di lapangan.

Saya juga belajar banyak tentang pengujian berbasis skenario nyata — setiap fitur diuji dalam konteks workflow aktual, bukan hanya unit test, untuk memastikan aplikasi benar-benar membantu pengguna di lapangan.
